## What Was Built

### 1. Line Detail Page — `/stats/lines/[slug]` (Stages 1–3)

The single biggest feature shipped: a dedicated analytics page for every F-line and D-pair with ≥150 min of 5v5 TOI.

**Stage 1 — Foundation & Identity**
- Static generation via `getStaticPaths()` from `lines.json` (zero runtime cost)
- Slug format: `{team}-{sorted-lastnames}-{season}-{game-type}` (e.g., `mtl-caufield-slafkovsky-suzuki-2025-26-playoffs`)
- Masthead: team logo + full player strip with headshots (via CORS proxy)
- Filter context bar showing opponent, season, game type
- 9-cell KPI strip: GP, TOI, xGF%, xGF, xGA, GF%, GF, GA, PDO
- PNG export button using canvas-direct renderer

**Stage 2 — Stat Breakdown Blocks**
- 6 full stat breakdown panels: CORSI, FENWICK, SHOTS, xGOALS, GOALS, PDO
- Each block shows FOR/AGAINST/PCT with threshold color coding (red/yellow/green)
- PDO computed as `sh_pct + sv_pct` (100-base); all fields sourced from enriched `lines.json`
- Pipeline work in `export_lines_json.py`: added `_compute_shot_stats()` to join shots + on_ice_rosters for FF/FA/SF/SA per line; CF/CA already in rapm_shifts

**Stage 3 — Shot Map (heat map rinks)**
- Two SVG half-rinks: shots for (left) and shots against (right)
- Per-shot circles sized by xG value, color-coded by outcome (goal = filled, SOG = ring)
- All shots normalized to attack-right direction
- Data flows from new `export_line_shots.py` → R2 → `/v1/lines/:slug/shots` → client fetch
- Background agent built and deployed this pipeline overnight

**Lines table → detail navigation**
- `/stats/lines` table rows are now clickable full-row links to `/stats/lines/{slug}`
- Multi-team filter: supports selecting up to 4 teams simultaneously (Option B with limit)
- Slug derivation in TypeScript: `src/lib/line-slug.ts` mirrors Python slug logic exactly

## Issues Encountered

1. **`final_state = 'FINAL'` vs `'OFF'`** — `build_rapm_shifts.py` only processes `final_state='OFF'` games. Tonight's game showed as FINAL. Required manual `UPDATE games SET final_state='OFF' WHERE game_id='2025030311'` + re-run. Normal pipeline would catch this next morning.

2. **Dashboard wrong Python/file paths** — `python3` on hgb-prod didn't have Flask. Venv at `/home/hgb/hockeygamebot/venv/bin/python`, file at `/home/hgb/hockeygamebot/dashboard.py`. Had been working from `/root/` assumption.

3. **Shell heredoc triple-quoting for server patches** — SSH inline heredoc with nested quotes failed. Fixed by: write Python patch file locally → `scp` to server → run remotely. More reliable for complex multi-line patches.

4. **NHL CDN headshot CORS** — Canvas taints when drawing cross-origin images without CORS headers. NHL CDN (`assets.nhle.com`) returns no `Access-Control-Allow-Origin`. Resolved with server-side proxy endpoint in hgb-api.

5. **`block.for` reserved word** — TypeScript interface used `for` property → renamed to `forVal`. Caught at build time.

6. **JSX `>=`/`<=` inside expressions** — Astro parser treated `>=` as closing tag. Fixed by extracting to helper function.

## Concerns / Technical Debt

1. **xG model divergence** — GSAx on the game page uses XgModelV3 (live bot, AUC 0.73); the analytics pipeline uses XGBoost (AUC 0.77). Users will see different GSAx numbers between game page and nightly stats. Labeled in section meta. Needs off-season unification via `hgb-shared`.

2. **`export_line_shots.py` needs Hetzner backfill** — The script was built but the backfill (`python scripts/export_line_shots.py --season 20252026`) hasn't been run on Hetzner yet. Shot maps on line detail pages will load empty until this runs + uploads.

3. **`nightly_pipeline.py` missing `export_line_shots.py` step** — The game-lines step was added, but `export_line_shots.py --all-recent` was not added. Needs one more step added to the pipeline for daily shot map updates.

4. **Static page count** — We now generate ~750+ static pages (32 teams + ~720+ line detail pages). Build time ~45–60s, total output ~400MB. Still well within Cloudflare Pages limits but worth monitoring as lines data grows.

5. **Team Stage 2 (opponent filter) deferred** — The `?opponent={abbr}` head-to-head feature requires `team_game_stats.json` export enrichment + `playerGameMap` data. ~4 hours of work. High business value for series previews.

## Enhancement Ideas

1. **Line detail: rolling form chart** — A sparkline showing xGF% over the last 5/10 games for this combination would answer "are they trending up?" Much more actionable than season averages.

2. **Shot map: filter by period** — The current map is all-game. A period-1 / period-3 split would reveal how lines perform under different game states (leading vs trailing).

3. **Team page: division standings mini-table** — When you land on `/teams/njd`, showing NJD's position in the Metro Division with 3 rows above/below would add enormous context without much data.

4. **Line detail: opponent breakdown** — Which team gave this line the most trouble? xGF% vs each opponent as a sortable mini-table. Directly answers the series prep use case.

5. **Share image auto-generation** — The `og:image` for line detail pages is currently the default HGB banner. A dynamic OG image showing the line's key stats would dramatically improve social sharing (the URL is the shareable artifact and it should look great unfurled).

6. **Combine line shots heat map with xG contours** — Overlay xG density contours on the shot map (from the entire league's shot model, not just this line) to show whether this line's shot locations are above or below expected. JFresh-style but for lines.

### Form Sparkline (added post-sprint, same night)

Rolling xGF% sparkline added to every line detail page. Commits: `2b72fcc` (analytics), `2bbde27` (site).

**Chart design:**
- SVG `viewBox="0 0 1000 72"` with `preserveAspectRatio="none"` — stretches to container width, dots stay circular
- Y axis: 25 / 50 / 75% reference lines + labels. 50% = league-average reference (dashed, slightly darker)
- Segment color: green (`#22c55e`) when both endpoints ≥ 50%; red (`#ef4444`) when both below; grey when crossing 50%
- Dots: 3px circles, white stroke, same green/red color logic. Invisible 14px hit targets for tooltip
- Tooltip: `YYYY-MM-DD · vs OPP · XX.X% xGF · X.Xm (GF–GA)`
- Threshold: only renders if `game_log.length >= 2`
