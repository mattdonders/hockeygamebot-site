# GameBot Social Runbook

Date: 2026-06-28

Purpose: a day-to-day operating checklist for using GameBot artifacts on social. The strategy lives in `docs/hgb-voice-growth-plan.md`; this is the short version to use when news drops or a hockey conversation is already moving.

## 1. Before Posting

Ask these in order:

1. Is there real hockey context?
   - trade rumor
   - signing/extension
   - fanbase debate
   - playoff/game moment
   - award/ranking discussion
   - request from a fan/community

2. Is there a useful artifact?
   - player card
   - goalie card
   - shotmap card
   - RAPM/history card
   - comparison card
   - team snapshot
   - game/bot artifact

3. Does the artifact add something the original post does not?
   - profile context
   - style/role read
   - team fit
   - trend
   - caveat
   - visual context point without saying "proof"

4. Which account should act first?
   - GameBot account: recurring format, broad hockey relevance, official artifact source
   - personal account: Devils/fanbase angle, human opinion, conversational post
   - team bot: fast game/team automation

5. What action fits?
   - reply: enter an existing large conversation
   - quote-post: create a timeline post around the artifact
   - original post: planned GameBot format or standalone artifact
   - no post: artifact is weak, stale, forced, or only repeats the news

## 2. Account Rules

### GameBot Account

Use for:

- official artifact originals
- recurring formats
- broad league/player context
- clean replies under large hockey news posts
- methodology-lite when needed

Avoid:

- sounding smug
- posting weak artifacts just to stay active
- asking an empty room for replies
- acting like the model is the final authority

### Personal Account

Use for:

- Devils-specific discussion
- human opinion
- quote-posting selected GameBot posts
- fanbase conversations where your voice/reach matters
- build-in-public updates

Avoid:

- turning every post into GameBot amplification
- making GameBot look smug by association
- assuming personal reach applies equally to every fanbase

### Team Bots

Use for:

- fast game/team updates
- additive context fans would not get from a score app without looking
- charts, scores, model signals, game-state artifacts

Avoid:

- ordinary low-value play-by-play
- over-editorializing
- high-cost Twitter volume before the bot posting audit

## 3. Reply Vs Quote

Reply when:

- the original post already has attention
- the artifact directly answers or improves the conversation
- discovery inside the replies matters
- GameBot account needs exposure to non-followers

Quote-post when:

- the artifact should live on your feed
- you have a real take
- the post helps train people to follow GameBot
- the artifact is strong enough to stand alone

Best growth workflow:

1. GameBot replies to the large/relevant post with the clean artifact.
2. Personal account quote-posts the GameBot reply only if there is a real take or fanbase angle.

Guardrail:

Do not quote-post every GameBot reply. Some replies should stay replies. Some takes should originate from personal. Some artifacts should originate from GameBot.

## 4. Player News Runbook

Use when:

- player is in rumors
- player signs or extends
- player changes teams
- fanbase debates whether the player fits
- large account posts player news

Artifact priority:

1. player card
2. shotmap card if scoring/location/style is the point
3. RAPM/history card if multi-season profile matters
4. comparison card if the debate is player-vs-player
5. trade/signing dossier when built

Default actions:

- Broad league news: GameBot reply or quote-post.
- Devils-specific angle: personal account can post directly.
- Large account news post: GameBot reply first; personal quote only if useful.
- Planned recurring format: GameBot original.

Copy shape:

`{Player} context: {profile read}. Card/profile: {url}`

Examples:

- `Vladar context: recent profile is stronger than the surface reputation, but contract/team context matters. Card: {url}`
- `DeBrincat fit read: real shooting/scoring value. The question is whether the rest of the profile fits the role/cost. Card: {url}`

Avoid posting when:

- the card does not add context
- the player data is stale or incomplete
- the only angle is dunking
- contract/value claim needs data you do not have yet

## 5. Team / Fanbase Debate Runbook

Use when:

- a fanbase is arguing about roster needs
- a team is linked to multiple players
- a player fit discussion is already moving
- team-vs-opponent context matters

Artifact priority:

1. player card with team-fit copy
2. team snapshot
3. line chemistry card
4. team-vs-opponent view
5. comparison card

Default actions:

- Devils conversation: personal account likely acts first.
- Non-Devils large-account thread: GameBot reply may be better.
- Threads Community: tailor copy to the community.
- Discord: artifact must stand alone without thread context.

Copy shape:

`I think he fits {team} because {hockey reason}. The card supports that with {profile signal}, but the context flag is {concern}.`

Avoid posting when:

- the fit take is mostly cap workflow speculation
- the artifact does not help the conversation
- the fanbase angle is forced

## 6. Game Moment Runbook

Default:

Game artifacts belong to game/team bots.

Manual post only when:

- playoff/high-leverage moment
- unusually shareable turning point
- broader hockey conversation
- bot artifact explains something people are already debating

Artifact priority:

1. turning point
2. goalie save/xG swing
3. game tilted here
4. final recap

Avoid manual posting when:

- it is ordinary play-by-play
- the bot already handled it and there is no broader conversation
- the artifact is only interesting because it exists

## 7. Copy Rules

Do:

- lead with hockey profile/context
- let the card carry most numbers
- use one number only when it materially clarifies the point
- add caveats without draining the post
- sound human, direct, and useful

Avoid:

- "proved"
- "receipts"
- "we told you"
- "debate over"
- "objectively better"
- "fraud"
- "washed" as GameBot language
- "HGB hates him"
- "AI says"
- "the model knows"
- "must trade for"
- "awful contract" unless value model supports it
- implying inside information
- pretending generated profile text is human scouting

Good default:

`Profile first. Card carries the numbers. Copy adds context.`

## 8. Weekly Review

Once per week, check:

- Which artifacts were shared/replied to?
- Which posts drove profile/page clicks?
- Which account originated the best posts?
- Did GameBot gain followers from replies/originals?
- Did any copy feel too robotic or too cautious?
- Which moments should become repeatable formats?

Keep:

- formats that generated replies, reposts, saves, Discord shares, or page visits

Cut:

- formats that needed explanation, felt forced, or got ignored repeatedly

## 9. First Walkthrough

When news drops, walk through this sequence:

1. Identify the context.
2. Pick the artifact.
3. Decide GameBot vs personal vs bot.
4. Decide reply vs quote vs original.
5. Write profile-first copy.
6. Post.
7. If useful, amplify from personal.
8. Capture what happened for the weekly review.
